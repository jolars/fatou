# JuliaFormatter formatter oracle blocked list — slugs deliberately diverging
# from JuliaFormatter (DefaultStyle) or not yet at parity, each with a one-line
# rationale. A blocked case is a known divergence, not a passing test.
# `allowlist ∪ blocked` covers the whole corpus.
#
# This list was re-baselined when the oracle target flipped from Runic.jl to
# JuliaFormatter.jl. The dominant finding: JuliaFormatter DefaultStyle is largely
# *preservation*-oriented — it does not canonicalize operator/assignment spacing
# or numeric literals — whereas Fatou is deterministic (Tenet 1). So most entries
# below are pending rule work or deliberate Fatou canonicalizations that the new
# target leaves to the user. They graduate to the allowlist as rules are flipped
# (or are reaffirmed as Tenet-1 divergences) via the `formatter-parity` skill.

# --- Tenet 1 (deliberate, permanent): JuliaFormatter preserves the user's
# operator/assignment spacing (`x=1`, `a*b`, `a==b`, `x->y`, `a&&b`, `return x+1`);
# Fatou canonicalizes to a single deterministic form so semantically-equivalent
# inputs format identically. Preserving would make output depend on input — Tenet 1
# forbids it. Will not be flipped to parity.
assignment_spacing
binary_spacing
comparison_chains
arrow_functions
logical_tight_divergence
keyword_statements
function_blocks
let_blocks

# --- Tenet 1 (deliberate, permanent): JuliaFormatter preserves numeric-literal
# spelling (`0xF`, `1E10`, `007.50`); Fatou normalizes hex byte-width padding and
# float form to one deterministic spelling. Will not be flipped to parity.
hex_literals
float_literals

# --- Tenet 1 (deliberate, permanent): JuliaFormatter's `for`-binding operator is
# input-dependent (preserves `∈`, uses `=` for a range iterable and `in` otherwise),
# so semantically-identical loops format differently by glyph/iterable. Fatou
# canonicalizes the iteration operator to a single `in` so they format identically.
# Will not be flipped to parity.
comprehension_for_in

# --- Wrong rule: JuliaFormatter packs comma-separated type params tight
# (`Dict{A,B}`); Fatou spaces them (Runic-shaped). Pending.
curly_type_params

# --- Missing/wrong rule: block, module, struct, ternary, where, and continuation
# layout (indentation, `;`-statement splitting, line breaks) not yet at parity
# with JuliaFormatter. Pending.
abstract_types
primitive_types
struct_blocks
control_flow
if_blocks
loop_blocks
begin_quote_blocks
module_baremodule
module_blocks
module_leading_comment
module_siblings
where_clauses
ternary_multiline
ternary_paren_branch
ternary_spacing
binary_continuation
paren_blocks
paren_multiline
block_comments
trailing_whitespace

# --- Missing/wrong rule: blank-line handling inside brackets, matrices, and
# block bodies differs from JuliaFormatter. Pending.
bracket_blank_lines
bracket_gap_blank_lines
matrix_blank_lines
matrix_gap_blank_lines
block_comments_in_blocks

# --- Missing/wrong rule: bracket and matrix item/comment layout (per-item breaks,
# trailing commas, inline comment placement) differs from JuliaFormatter. Pending.
matrices
multiline_brackets
multiline_matrices
bracket_comments
bracket_block_comments
matrix_block_comments

# --- Comment spacing: JuliaFormatter's pre-comment whitespace rule differs from
# Fatou's one-space canonicalization (Tenet 1 / pending). E.g. it keeps no space
# before a trailing `#=`, and preserves multi-space before line comments.
block_comment_spacing_divergence
bracket_comment_spacing_divergence
trailing_comment_spacing_divergence
trailing_comments
