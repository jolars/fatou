# Runic formatter oracle blocked list — slugs deliberately diverging from Runic
# or not yet supported, each with a one-line rationale. A blocked case is a known
# divergence, not a passing test. `allowlist ∪ blocked` covers the whole corpus.
#
# ── FOUNDATIONAL DIVERGENCE: line breaking is not shared ──────────────────────
# Runic is NOT line-width-sensitive. It *preserves* the user's line breaks and
# normalizes only the spacing and indentation within whatever layout the source
# already chose. Fatou is the opposite (Tenet 1): it *fully reflows*, laying out
# each construct from scratch under `line_width`, ignoring source breaks and user
# whitespace entirely. Same tokens => same output, regardless of input form.
#
# Consequence for this oracle: for any construct whose layout depends on a source
# break, `format(input)` and `runic(input)` diverge in BOTH directions — Fatou
# collapses user breaks that fit within `line_width`, and breaks lines that
# exceed it where Runic leaves them long. These are deliberate Tenet-1
# divergences, not parity gaps, and Runic-pinned `expected.jl` is the WRONG
# target for them. Until Fatou's reflow engine lands, some fixtures pass only
# because Fatou currently still mirrors Runic's break-preservation ("false
# parity"); those are being audited out (see RECAP.md) and either re-minted to a
# Fatou-authored `expected.jl` or recorded here.

# Tenet 1 divergence: Runic *preserves* the user's spacing around `&&`/`||`
# (it normalizes neither `a&&b` nor `a && b`). Fatou must be deterministic, so it
# canonicalizes them as spaced; tight-written input therefore diverges.
logical_tight_divergence

# Runic inserts an explicit `return` before a function's trailing block
# expression. Return-insertion is not implemented (it is a semantic rewrite, not
# a layout rule); deferred.
control_flow

# Tenet 1 divergence: Runic *preserves* the user's pre-`#` whitespace on a
# trailing comment (≥1 space verbatim). Fatou must be deterministic, so it
# canonicalizes to exactly one space; multi-space input therefore diverges.
trailing_comment_spacing_divergence

# Tenet 1 divergence (same as above, inside a broken bracket): Runic preserves
# the user's pre-`#` whitespace on a trailing/own-line item comment; Fatou
# canonicalizes to one space, so multi-space input diverges. (Matrix interiors
# are kept verbatim, so matrix comments match Runic and need no entry.)
bracket_comment_spacing_divergence

# Tenet 1 divergence (same as above, block comment in a block body): Runic
# preserves the user's pre-`#=` whitespace on a trailing block comment; Fatou
# canonicalizes to one space, so multi-space input diverges. (Own-line and
# multi-line block comments match Runic and need no entry.)
block_comment_spacing_divergence

# Deliberate divergence: Runic preserves blank lines inside a parenthesized
# expression (capped at two). A single parenthesized value gains nothing from
# interior blanks, so Fatou keeps the source-driven break but strips the blanks
# (tight `(` / `+4` body / `)`); blank-padded input therefore diverges.
paren_blank_line_divergence
